# Roadmap de implementación — ProyectaDi

## Estado general

ProyectaDi se desarrollará mediante sprints cortos, verificables y documentados. Cada sprint deberá producir un avance funcional o estructural que pueda revisarse, probarse y subirse al repositorio.

## Sprints del proyecto

| Sprint | Enfoque | Resultado esperado |
|---|---|---|
| Sprint 0 | Preparación del repositorio, documentación y entorno | Estructura base, README, docs y configuración inicial |
| Sprint 1 | Backend base con FastAPI | API mínima funcionando con health check y configuración |
| Sprint 2 | Base de datos y modelos principales | Usuarios, proyectos, etapas y tareas persistentes |
| Sprint 3 | Autenticación y roles | Registro, login y control básico de permisos |
| Sprint 4 | Frontend base | Interfaz React con navegación principal y dashboard inicial |
| Sprint 5 | Avances y presupuesto | Bitácora de avances, gastos y cálculos automáticos |
| Sprint 6 | Gráficas y reportes | Indicadores visuales e informe PDF básico |
| Sprint 7 | Colaboración y evidencias | Invitaciones, archivos y permisos por proyecto |
| Sprint 8 | Pruebas, documentación y versión beta | Manual de usuario, pruebas y entrega beta |
| Sprint 9 | Empaquetado de escritorio | App instalable con Tauri conectada al backend |

## Criterio para cerrar Sprint 0

Sprint 0 se considerará completo cuando el repositorio tenga:

- estructura base del proyecto;
- ambiente virtual del backend creado;
- carpeta preparada para frontend;
- documentación inicial;
- archivo README inicial;
- archivo .gitignore;
- repositorio conectado a GitHub;
- primer commit subido correctamente.
